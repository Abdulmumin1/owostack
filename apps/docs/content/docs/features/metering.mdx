# Usage Metering

Owostack provides a high-performance, atomic usage metering system powered by **Cloudflare Durable Objects**. This allows you to track high-concurrency events (like API calls) without race conditions.

## How it Works

1.  **Configure Feature**: Set a feature type to `metered` in your Plan.
2.  **Set Limits**: Define the `limit` (e.g., 500) and `reset_interval` (e.g., "monthly").
3.  **Track Usage**: Call the SDK to record usage events.

## Tracking usage

Use the `track()` method to increment usage.

```ts
await owo.track({
  customer: "user_123",
  feature: "api_calls",
  value: 1, // Default is 1
});
```

The system automatically:

- Checks if the user has remaining quota.
- Atomically increments the usage counter.
- Updates the balance.
- Returns the new status.

## Configuration Options

When defining a metered feature in a Plan, you can configure:

### Reset interval

How often the usage counter resets.

- `monthly`: Resets on the subscription billing cycle anchor.
- `weekly`: Resets every week.
- `daily`: Resets every day.
- `never`: Usage accumulates indefinitely (good for lifetime limits).

### Rollover

Allow unused usage to carry over to the next period.

- **Rollover Enabled**: Unused balance is added to the next period's limit.
- **Max Rollover**: Cap the amount that can be rolled over (e.g., "Max 2x monthly limit").

### Usage model

- `included`: Usage up to the limit is included in the plan base price.
- `usage_based`: All usage is billable (no included allowance).
- `prepaid`: Usage consumes from a prepaid credit balance.

Overages are controlled separately via the `overage` setting on the plan feature (`block` or `charge`).

## Lazy configuration

Owostack uses a "lazy configuration" pattern. You don't need to manually provision meters for every user.
When you first call `track()` for a user, the system automatically initializes their meter based on their current plan configuration.
