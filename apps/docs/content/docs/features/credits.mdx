# Credits

The Credits system allows you to implement prepaid billing models, often used for AI applications, API marketplaces, or "top-up" wallets.

## Concept

Instead of simple counters (1 API call = 1 unit), you can abstract usage into **Credits**.
Users purchase credit packs (e.g., $10 for 1000 credits), and features consume credits at different rates.

## Credit Cost

You can assign a **Credit Cost** to each feature in a Plan.

| Feature         | Credit Cost |
| :-------------- | :---------- |
| `gpt-3.5-turbo` | 1 credit    |
| `gpt-4`         | 20 credits  |
| `image_gen`     | 50 credits  |

## Consuming Credits

When you call `track()`, if the feature has a `credit_cost`, Owostack automatically deducts the calculated amount from the user's credit balance.

```ts
// Tracking 1 unit of 'gpt-4' (Cost: 20 credits)
await owo.track({
  customer: "user_123",
  feature: "gpt-4",
});
// Deducts 20 credits from balance
```

## Adding credits

Credits are typically added when a customer purchases a credit-pack add-on. Configure an add-on plan with a credit feature and Owostack will apply the balance when the payment succeeds.

## Monitoring balances

Use `check()` to show remaining balances in your UI and `track()` to decrement credits for usage-based actions.
