# Credits

The Credits system allows you to implement prepaid billing models, often used for AI applications, API marketplaces, or "top-up" wallets.

## Concept

Instead of simple counters (1 API call = 1 unit), you can abstract usage into **Credits**.
Users purchase credit packs (e.g., $10 for 1000 credits), and features consume credits at different rates.

## Credit Systems

Owostack allows you to define **Credit Systems** that link multiple features to a single shared balance.

```ts
import { creditSystem, metered } from "@owostack/core";

const gpt4 = metered("gpt-4");
const dallE = metered("dall-e");

// Define a system where different features cost different amounts of the same credits
export const aiCredits = creditSystem("ai-credits", {
  name: "AI Credits",
  features: [
    gpt4(20), // 1 GPT-4 call costs 20 credits
    dallE(50), // 1 Dall-E generation costs 50 credits
  ],
});
```

## Assigning Credits to Plans

Once defined, you can include credits in your plans:

```ts
plan("pro", {
  // ...
  features: [
    aiCredits.credits(1000), // 1,000 credits included per month
  ],
});
```

## Consuming Credits

When you call `track()`, if the feature belongs to a credit system, Owostack automatically deducts the calculated amount from the user's credit balance.

```ts
// Tracking 1 unit of 'gpt-4' (Cost: 20 credits)
await owo.track({
  customer: "user_123",
  feature: "gpt-4",
});
// Deducts 20 credits from 'ai-credits' balance
```

## Monitoring balances

Use `check()` to see if a user has enough credits remaining for an action.

```ts
const { allowed, remaining } = await owo.check({
  customer: "user_123",
  feature: "gpt-4",
});

if (!allowed) {
  console.log("Not enough credits!");
}
```
