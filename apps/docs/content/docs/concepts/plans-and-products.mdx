# Plans & products

A **plan** is what a customer subscribes to. It maps to a provider plan object and defines the feature set a customer should have.

## Plan groups (mutual exclusivity)

Plans in the same **plan group** are mutually exclusive.

- Switching within a group is treated as a *plan switch*.
- Switching across groups is treated as a *new subscription*.

## Add-ons

Add-ons are products that stack on top of a base plan.

- Add-ons do not replace the base plan.
- Add-ons can be used for credit packs, extra seats, or additional limits.

## Updating plans

Changing a plan’s features affects entitlements immediately for customers on that plan.
