# API Reference

Base URL: `https://api.owostack.dev`

Public API paths are available at both `/v1/*` and `/api/v1/*`.

All endpoints require authentication via API key in the `Authorization` header.

---

## Authentication

```bash
Authorization: Bearer owo_sk_your_api_key
```

---

## POST /v1/attach

Subscribe a customer to a plan.

### Request

```json
{
  "customer": "user_123",
  "product": "pro-monthly",
  "customerData": { "email": "user@example.com" },
  "metadata": {},
  "provider": "{provider-id}",
  "callbackUrl": "https://yourapp.com/billing"
}
```

| Field          | Type   | Required | Description                         |
| -------------- | ------ | -------- | ----------------------------------- |
| `customer`     | string | ✅       | Your internal user ID               |
| `product`      | string | ✅       | Plan slug or ID                     |
| `customerData` | object | -        | Auto-create customer payload        |
| `metadata`     | object | -        | Custom metadata                     |
| `provider`     | string | -        | Optional provider override          |
| `callbackUrl`  | string | -        | Redirect URL after payment          |
| `currency`     | string | -        | Override currency (ISO 4217)        |
| `channels`     | string[] | -      | Provider payment channels           |
| `region`       | string | -        | Region selector for provider routing |

### Response

```json
{
  "success": true,
  "requiresCheckout": true,
  "checkoutUrl": "https://provider.example/checkout",
  "type": "new",
  "message": "Checkout created for Pro",
  "subscriptionId": "sub_123"
}
```

## POST /v1/check

Check customer entitlements for a feature.

### Request

```json
{
  "customer": "user_123",
  "feature": "api_calls",
  "value": 1,
  "entity": "org_acme",
  "sendEvent": true
}
```

| Field          | Type   | Required | Description                         |
| -------------- | ------ | -------- | ----------------------------------- |
| `customer`     | string | ✅       | Your internal user ID               |
| `feature`      | string | ✅       | Feature key                          |
| `value`        | number | -        | Amount to check (default: 1)         |
| `entity`       | string | -        | Scope usage to an entity (seat/org)  |
| `sendEvent`    | boolean | -       | Atomically track if allowed          |
| `customerData` | object | -        | Auto-create customer payload         |

### Response

```json
{
  "allowed": true,
  "code": "access_granted",
  "usage": 500,
  "limit": 10000,
  "balance": 9500,
  "resetsAt": "2024-06-01T00:00:00Z",
  "resetInterval": "monthly",
  "details": { "message": "Access granted — used 500 of 10000." }
}
```

## POST /v1/track

Record usage for metered billing.

### Request

```json
{
  "customer": "user_123",
  "feature": "api_calls",
  "value": 1,
  "entity": "seat_456",
  "metadata": {}
}
```

| Field          | Type   | Required | Description                         |
| -------------- | ------ | -------- | ----------------------------------- |
| `customer`     | string | ✅       | Your internal user ID               |
| `feature`      | string | ✅       | Feature key                          |
| `value`        | number | -        | Usage amount (default: 1)            |
| `entity`       | string | -        | Scope usage to an entity (seat/org)  |
| `metadata`     | object | -        | Event metadata                        |
| `customerData` | object | -        | Auto-create customer payload          |

### Response

```json
{
  "success": true,
  "allowed": true,
  "code": "tracked",
  "usage": 501,
  "limit": 10000,
  "balance": 9499,
  "resetsAt": "2024-06-01T00:00:00Z",
  "resetInterval": "monthly",
  "details": { "message": "Usage tracked successfully (9499 remaining)." }
}
```

## POST /v1/addon

Purchase a credit pack add-on.

### Request

```json
{
  "customer": "user_123",
  "pack": "500-credits",
  "quantity": 1
}
```

| Field      | Type   | Required | Description                          |
| ---------- | ------ | -------- | ------------------------------------ |
| `customer` | string | ✅       | User ID or email                     |
| `pack`     | string | ✅       | Credit pack slug or ID               |
| `quantity` | number | -        | Number of packs to buy (default: 1) |

### Response

```json
{
  "success": true,
  "requiresCheckout": false,
  "balance": 1500,
  "creditsAdded": 500
}
```

If the customer has no card on file:

```json
{
  "success": true,
  "requiresCheckout": true,
  "checkoutUrl": "https://provider.example/checkout"
}
```

## POST /v1/wallet/setup

Initiate card authorization. Returns a URL for the customer to complete card setup.

### Request

```json
{
  "customer": "user_123",
  "callbackUrl": "https://yourapp.com/billing",
  "provider": "paystack"
}
```

| Field         | Type   | Required | Description                    |
| ------------- | ------ | -------- | ------------------------------ |
| `customer`    | string | ✅       | User ID or email               |
| `callbackUrl` | string | -        | Redirect URL after setup       |
| `provider`    | string | -        | Provider ID override           |

### Response

```json
{
  "url": "https://provider.example/authorize",
  "reference": "ref_xyz789"
}
```

## POST /v1/sync

Push a catalog of features and plans from code. Used by the CLI and `owo.sync()`.

### Request

```json
{
  "features": [
    { "slug": "api-calls", "name": "API Calls", "type": "metered" }
  ],
  "plans": [
    {
      "slug": "pro",
      "name": "Pro",
      "price": 2000,
      "currency": "USD",
      "interval": "monthly",
      "features": [
        { "featureSlug": "api-calls", "enabled": true, "limit": 10000, "resetInterval": "monthly" }
      ]
    }
  ]
}
```

### Response

```json
{
  "success": true,
  "features": { "created": ["api-calls"], "updated": [], "unchanged": [] },
  "plans": { "created": ["pro"], "updated": [], "unchanged": [] },
  "warnings": []
}
```

## Error Responses

All errors follow this format:

```json
{
  "success": false,
  "error": {
    "code": "ErrorCode",
    "message": "Human-readable message",
    "details": {}
  }
}
```

### Error Codes

| Code              | HTTP Status | Description                |
| ----------------- | ----------- | -------------------------- |
| `AuthError`       | 401         | Invalid or missing API key |
| `ValidationError` | 400         | Invalid request body       |
| `NotFoundError`   | 404         | Resource not found         |
| `DatabaseError`   | 500         | Internal database error    |

## Webhooks

Owostack receives webhooks from providers at:

```
POST /webhooks/{organizationId}/{providerId}
```

Configure this in your provider dashboard. See [Webhook Setup](/docs/webhook-setup) for details.
