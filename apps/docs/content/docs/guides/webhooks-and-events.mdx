# Webhooks & events

Webhooks are how Owostack stays in sync with your payment provider.

## Verification

Owostack verifies webhook payloads using your provider’s recommended method (for Paystack this is HMAC SHA-512). If verification fails, events are rejected.

## Processing

After verification, Owostack:

1. Parses the provider event into a normalized internal event.
2. Updates subscription/customer state.
3. Provisions entitlements and updates meters/credits when relevant.

## Common events

- Subscription created / enabled / disabled
- Payment succeeded / failed

## Debugging

If something looks off:

- Confirm your provider webhook secret is correct
- Confirm you’re sending events to the right organization webhook URL
- Check the dashboard events/logs for processing errors
