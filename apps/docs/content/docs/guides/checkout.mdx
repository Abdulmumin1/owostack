# Checkout

Owostack’s checkout flow is responsible for:

- Creating (or reusing) a customer
- Starting a provider checkout session when needed
- Ensuring provider webhooks can be mapped back to your organization/project/customer

## Subscriptions

Use `attach()` to subscribe a customer to a plan. If payment is required, Owostack returns a `checkoutUrl` for the provider.

```ts
const result = await owo.attach({
  customer: "user@example.com",
  plan: "pro-monthly",
});

if (result.checkoutUrl) {
  // Redirect to provider checkout
}
```

## When checkout is not required

Owostack may return an already-active subscription when:

- The customer is already subscribed to the requested plan
- The switch is lateral and can be applied immediately without payment
- The plan is free or trial-only

## Webhooks are required

Checkout only becomes “real” after webhooks confirm payment or subscription changes. Make sure your webhook URL includes the provider ID.

Make sure you’ve completed [Webhook Setup](/docs/webhook-setup).
